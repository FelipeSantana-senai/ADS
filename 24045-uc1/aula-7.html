Repetindo tarefas: laço enquanto (while)

Depois de aprender como tomar decisões com base em condições usando as estruturas condicionais, chegou a hora de explorar situações em que certas ações precisam ser repetidas diversas vezes. Nesta aula, você conhecerá o laço de repetição while, que permite executar comandos enquanto uma condição for verdadeira.

Muitos processos reais repetem ações, como sensores ou esteiras. O laço while permite repetir comandos enquanto uma condição for verdadeira.

Pense em uma esteira de produção: ela não para enquanto houver peças chegando. Ou imagine um sistema que monitora sensores de temperatura. Ele precisa ler os dados constantemente, não só uma vez.

Na programação, esse tipo de comportamento é possível graças às estruturas de repetição, e aqui surge um conceito muito importante: o laço “enquanto” (while).

Funciona assim: enquanto uma condição for considerada verdadeira, o sistema continuará repetindo a ação determinada. Assim que essa condição deixar de ser verdadeira, o sistema interrompe a execução.

Conheça na ilustração a seguir como esse processo ocorre passo a passo:

Verificar condição

O sistema avalia se a condição é verdadeira.

Condição verdadeira

A condição é determinada como verdadeira.

Executar ação

O sistema executa a ação especificada.

Repetir ação

O sistema repete a ação enquanto a condição permanecer verdadeira.

Condição falsa

A condição é determinada como falsa.

Parar ação

O sistema para de executar a ação.

Veja exemplos práticos em que isso ocorre:

Continuar vendendo.

Manter o motor ligado.

Continuar processando.

Além disso, comandos como break e continue ajudam a controlar essas repetições:

Serve para interromper a repetição quando algo específico acontece.

Faz o sistema pular uma execução específica e seguir para a próxima.

Sem essa lógica, seria impossível automatizar processos, monitorar dados em tempo real ou fazer sistemas executarem tarefas de forma contínua e eficiente.

Agora que você compreendeu o funcionamento do laço while e sua aplicação em cenários repetitivos, é hora de ver esse conhecimento sendo aplicado na prática.
